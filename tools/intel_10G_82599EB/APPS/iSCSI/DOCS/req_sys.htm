System Requirements

Before installing Intel® Server Adapters and Intel® Ethernet iSCSI Boot firmware, check your system for the following minimum configuration requirements. If you are using an Intel Ethernet iSCSI Boot-capable OEM server system, refer to the instructions provided from the OEM to enable Intel Ethernet iSCSI Boot.

System Hardware Compatibility for Network Adapters

  • One of the following slot types, depending on your adapter:
    • A PCI-Express slot (v1.0a or newer), 1x, 4x, 8x or 16x
  • System memory 128 MB of RAM minimum required; 256 MB or more recommended.
  • The latest BIOS for your computer.

Supported 32-bit Operating Systems

Intel® Ethernet iSCSI Boot is supported on remote booting of the following IA-32 operating systems:

  • Red Hat* Enterprise Linux 4 U3, Red Hat* Enterprise Linux 5.x
  • Novell SUSE* Linux Enterprise 9 SP3, Novell SUSE* Linux Enterprise 10 or later

Platform Requirements for Intel® 64 Architecture

Intel® Ethenet iSCSI Boot is supported on remote booting of the following Intel® 64 operating systems:

  • Microsoft Windows Server 2008 R2, with latest service pack
  • Red Hat Enterprise Linux 4 U3
  • Red Hat Enterprise Linux 5.x
  • Novell SUSE Linux Enterprise 9 SP3, 10 or later

A platform that supports Intel® 64 will run in either 64-bit mode or 32-bit compatibility mode. In order for it to run in 64-bit mode, the following requirements must be met:

  • The Intel® 64 system must have a 64-bit BIOS that takes advantage of Intel® 64 Architecture
  • One of the supported 64-bit operating systems must be installed.

The installer for the adapter drivers will only list the drivers that are compatible with the currently running operating system.

  • If the system is running in compatibility mode, only IA-32 drivers will be available.
  • If the system is running in 64-bit mode, only Intel® 64 drivers will be available.
